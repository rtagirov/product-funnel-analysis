# Product Funnel Analysis

## Цель проекта
Провести анализ продуктовой воронки и посчитать конверсии между ключевыми этапами:
визит → регистрация → пробный период → покупка.

Проект демонстрирует навыки продуктовой аналитики и работы с событийными данными.

---

## Описание данных
Используется событийный датасет, имитирующий поведение пользователей SaaS-продукта.

Каждая строка — отдельное событие пользователя и содержит:
- user_id — идентификатор пользователя
- timestamp — время события
- event — тип события (visit, signup, trial_start, purchase)
- source — источник трафика

Датасет используется для демонстрации аналитического подхода и расчёта бизнес-метрик.

---

## Этапы анализа

1. Предобработка данных  
   - проверка типов
   - анализ временного диапазона
   - проверка распределения событий на пользователя

2. Построение воронки  
   - агрегация событий по этапам
   - сегментация по источникам трафика

3. Расчёт конверсий  
   - visit → signup
   - signup → trial
   - trial → purchase

---

## Ключевые метрики
- Конверсия между этапами воронки
- Сравнение эффективности источников трафика
- Выявление этапов с наибольшими потерями пользователей
