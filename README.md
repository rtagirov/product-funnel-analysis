# Анализ воронки продукта (SaaS Freemium, синтетические данные)

## О проекте

Cмоделировал поведение пользователей в воронкe цифрового продукта:

visit → signup → trial_start → purchase

Данные сгенерированы программно. Цель проекта отработать:

- построение воронки
- анализ потерь пользователей
- модель атрибуции (first touch)
- базовую юнит-экономику
- работу с событийным логом

---

## Что смоделировано

Сценарий: Рекламная кампания SaaS-продукта с бесплатной регистрацией и триалом.

Каждый пользователь:

- сначала попадает на сайт (visit)
- может зарегистрироваться (signup)
- может начать пробный период (trial_start)
- может совершить покупку (purchase)

Дополнительно:
- пользователь может совершать повторные визиты
- события строго идут по логике воронки (без пропусков шагов)
- каждому пользователю назначается источник первого касания (ads, organic, email, referral)

---

## Что было сделано

### 1. Генерация данных

- смоделирована логика поведения пользователей
- задана вероятность перехода между шагами
- добавлены повторные визиты
- сгенерирована выручка для покупок
- проверена целостность данных

---

### 2. Анализ событий

- подсчитано количество событий по дням
- проверено распределение типов событий
- визуализирована динамика рекламной кампании

---

### 3. Анализ воронки

- посчитано число пользователей на каждом этапе
- рассчитаны конверсии и потери
- построена waterfall-диаграмма
- определён шаг с наибольшей потерей пользователей

---

### 4. Анализ по источникам (First Touch Attribution)

Пользователи были закреплены за источником первого визита.

Для каждого канала рассчитано:

- количество привлечённых пользователей
- конверсия в покупку
- общая выручка
- средняя выручка на пользователя (ARPU)
- средний чек (ARPPU)

Это позволяет сравнить качество трафика.
